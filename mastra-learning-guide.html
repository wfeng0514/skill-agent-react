基于 TypeScript 构建 AI 驱动应用与智能体的生产级框架

📦 v1.0 🔤 TypeScript 🤖 600+ 模型 🔌 17+ 向量库

框架总览

Mastra 是一个基于现代 TypeScript 堆栈构建的 AI Agent 框架，定位从早期原型到生产就绪的全周期。可集成 React、Next.js、Node.js，也可作为独立服务器部署。

核心能力

🤖 Agent
使用 LLM + 工具解决开放式任务的自主智能体，支持记忆、推理、结构化输出
🔧 Tool
与外部 API/服务结构化交互，Zod Schema 校验输入输出
🔗 Workflow
基于图的工作流引擎，支持串行/并行/条件分支/暂停恢复
🧠 Memory
三层记忆：消息历史 + 工作记忆 + 语义召回
🔍 RAG
文档处理 + 分块嵌入 + 向量检索，17+ 向量数据库
📊 Observability
内置追踪、日志、脱敏，支持 Mastra Studio / Cloud / OTel

适用场景

场景 说明
对话式智能体 客服、入职引导、技术支持
领域专用助手 代码生成、法律咨询、金融分析
工作流自动化 DevOps、内容生成、销售流程
决策支持 数据分析、报告生成
嵌入式 Copilot 内部工具、SaaS 产品内的 AI 助手

快速开始

方式一：脚手架创建（推荐）

npm create mastra@latest

交互式引导，自动生成项目结构。

方式二：手动集成到已有项目

# 1. 初始化 Mastra
npx mastra@latest init

# 这会在 src/mastra/ 下生成：
# - index.ts        → Mastra 实例
# - tools/weather-tool.ts → 天气工具示例
# - agents/weather-agent.ts → 天气 Agent 示例

# 2. 安装前端依赖（如需聊天 UI）
npm install @mastra/ai-sdk@latest @ai-sdk/react ai

# 3. 初始化 AI Elements 组件库
npx ai-elements@latest
💡 提示：需要 Node.js v22.13.0+ 和模型提供商的 API Key（如 OpenAI、Anthropic、Google 等）。

Mastra 实例配置

import { Mastra } from "@mastra/core/mastra";
import { PinoLogger } from "@mastra/loggers";
import { LibSQLStore } from "@mastra/libsql";
import { DuckDBStore } from "@mastra/duckdb";
import { Observability, DefaultExporter, CloudExporter, SensitiveDataFilter } from "@mastra/observability";
import { chatRoute } from "@mastra/ai-sdk";

export const mastra = new Mastra({
  // 注册 Agent
  agents: { weatherAgent },

  // 注册 Workflow
  workflows: { weatherWorkflow },

  // 复合存储
  storage: new MastraCompositeStore({
    stores: [
      // 默认存储：SQLite（对话记忆等）
      new LibSQLStore({ id: "default", url: "file:./mastra.db" }),
      // 可观测性存储：DuckDB（Trace 数据）
      new DuckDBStore({ id: "observability", url: "md:mastra_observability" }),
    ],
  }),

  // 日志
  logger: new PinoLogger({ level: "info" }),

  // 可观测性
  observability: new Observability({
    configs: {
      default: {
        serviceName: "mastra",
        exporters: [
          new DefaultExporter(),  // 持久化到本地存储
          new CloudExporter(),    // 推送到 Mastra Cloud
        ],
        spanOutputProcessors: [
          new SensitiveDataFilter(),  // 自动脱敏
        ],
      },
    },
  }),

  // 自定义 API 路由
  server: {
    apiRoutes: [
      chatRoute({ path: "/chat/:agentId" }),
    ],
  },
});

Agent 智能体

Agent 是使用 LLM 和工具来解决开放式任务的实体。核心特征：

🎯 自主推理
推理目标并决定使用哪些工具
🧠 记忆保留
保留对话记忆，跨会话保持上下文
🔄 内部迭代
在发出最终答案前进行循环推理
📋 结构化响应
生成可在 UI 中渲染的结构化数据
🧩 组合性
可直接使用，也可组合成工作流/代理网络

基础创建

import { Agent } from "@mastra/core/agent";

export const myAgent = new Agent({
  id: "my-agent",
  name: "My Agent",
  instructions: "You are a helpful assistant.",
  model: "openai/gpt-5.1",      // 支持 600+ 模型
  tools: { myTool },             // 绑定工具
  memory: new Memory({...}),     // 绑定记忆
});

指令格式（3 种）

// 1. 字符串（最常见）
instructions: "You are a helpful assistant."

// 2. 字符串数组
instructions: ["You are a helpful assistant.", "Always be concise."]

// 3. 系统消息数组
instructions: [{ role: "system", content: "You are a helpful assistant." }]

动态指令

指令可设为异步函数，支持运行时根据用户上下文个性化提示：

instructions: async ({ requestContext }) => {
  const userName = requestContext.get("userName");
  return `You are an assistant for ${userName}. Be friendly and helpful.`;
}

动态模型选择

根据请求上下文动态切换模型：

model: async ({ requestContext }) => {
  const tier = requestContext.get("user-tier");
  return tier === "enterprise"
    ? "openai/gpt-5.1"
    : "openai/gpt-4.1-mini";
}

生成与流式

// 普通生成 — 等完整输出后返回
const response = await myAgent.generate([
  { role: "user", content: "Hello!" }
]);
console.log(response.text);

// 流式生成 — 实时返回 Token
const stream = await myAgent.stream("Tell me a story");
for await (const chunk of stream.textStream) {
  process.stdout.write(chunk);
}

步骤控制

const response = await myAgent.generate("...", {
  maxSteps: 5,  // 最大 LLM 调用次数（含工具执行）

  onStepFinish: ({ text, toolCalls, toolResults }) => {
    console.log("Step done:", text || toolCalls);
  },

  onFinish: ({ text, steps, finishReason, usage }) => {
    console.log("All done:", text);
    console.log("Token usage:", usage);
  },
});

结构化输出

import { z } from "zod";

const response = await myAgent.generate("Analyze this text", {
  output: z.object({
    sentiment: z.enum(["positive", "negative", "neutral"]),
    confidence: z.number(),
    keywords: z.array(z.string()),
  }),
});

console.log(response.object);  // { sentiment: "positive", confidence: 0.92, ... }

图像分析（多模态）

const response = await myAgent.generate([
  {
    role: "user",
    content: [
      { type: "text", text: "What's in this image?" },
      { type: "image", image: "https://example.com/photo.jpg" },
    ],
  },
]);

注册与引用

💡 最佳实践：在 Mastra 实例中注册 Agent，通过 mastra.getAgent() 获取引用，而非直接 import。这样 Agent 能访问共享资源（内存、日志、可观测性）。 
// 注册
export const mastra = new Mastra({ agents: { myAgent } });

// 引用（推荐）
const agent = mastra.getAgent("myAgent");

核心 API 速查

API 说明
new Agent({...}) 实例化代理
mastra.getAgent(id) 获取已注册代理引用
agent.generate(messages, opts) 同步生成完整响应
agent.stream(messages, opts) 流式生成响应
response.text 获取纯文本结果
response.object 获取结构化数据（需配合 Zod output）

Tool 工具

Tool 让 Agent 能与外部 API 和服务进行结构化交互，突破纯语言生成的限制。每个 Tool 都通过 Zod Schema 定义输入输出，确保类型安全。

定义工具

import { createTool } from "@mastra/core/tools";
import { z } from "zod";

export const weatherTool = createTool({
  id: "get-weather",
  description: "Get current weather for a location",
  inputSchema: z.object({
    location: z.string().describe("City name"),
  }),
  outputSchema: z.object({
    temperature: z.number(),
    humidity: z.number(),
    conditions: z.string(),
    location: z.string(),
  }),
  execute: async ({ context }) => {
    const { location } = context;
    // 调用外部 API...
    const data = await fetchWeatherData(location);
    return {
      temperature: data.temp,
      humidity: data.humidity,
      conditions: data.condition,
      location,
    };
  },
});

绑定到 Agent

export const myAgent = new Agent({
  id: "my-agent",
  name: "My Agent",
  instructions: "You are a weather assistant. Use the weather tool to get data.",
  model: "openai/gpt-5.1",
  tools: { weatherTool },  // 绑定
});

工具执行流程

用户请求 → LLM 决策调用 Tool → Schema 校验输入 → execute 执行 → Schema 校验输出 → 结果返回 LLM

在 Workflow 步骤中使用工具

import { createStep } from "@mastra/core/workflows";

const fetchWeatherStep = createStep({
  id: "fetch-weather",
  inputSchema: z.object({ city: z.string() }),
  outputSchema: z.object({ temperature: z.number(), conditions: z.string() }),
  execute: async ({ inputData }) => {
    // 直接调用工具的 execute 方法
    const result = await weatherTool.execute({ context: { location: inputData.city } });
    return { temperature: result.temperature, conditions: result.conditions };
  },
});
💡 设计原则：Tool 应该是纯函数式的——给定相同输入，始终返回相同输出。副作用（写数据库、发邮件等）应该明确标注，便于测试和调试。

Workflow 工作流

Workflow 允许使用清晰、结构化的步骤定义复杂任务序列，而非依赖单个 Agent 的推理。

核心原则

1. 第一个步骤的 inputSchema 必须与工作流的 inputSchema 匹配
2. 最后一个步骤的 outputSchema 必须与工作流的 outputSchema 匹配
3. 每个步骤的 outputSchema 必须与下一个步骤的 inputSchema 匹配（如不匹配，使用 .map() 转换）

定义步骤

import { createStep, createWorkflow } from "@mastra/core/workflows";
import { z } from "zod";

const step1 = createStep({
  id: "format-step",
  inputSchema: z.object({ message: z.string() }),
  outputSchema: z.object({ formatted: z.string() }),
  execute: async ({ inputData }) => {
    return { formatted: inputData.message.toUpperCase() };
  },
});

const step2 = createStep({
  id: "emphasize-step",
  inputSchema: z.object({ formatted: z.string() }),
  outputSchema: z.object({ emphasized: z.string() }),
  execute: async ({ inputData }) => {
    return { emphasized: `${inputData.formatted}!!!` };
  },
});

控制流

1. 串行 — .then()

export const workflow = createWorkflow({
  id: "my-workflow",
  inputSchema: z.object({ message: z.string() }),
  outputSchema: z.object({ emphasized: z.string() }),
})
  .then(step1)
  .then(step2)
  .commit();  // 别忘了 commit()

2. 并行 — .parallel()

// 并行步骤的输出结构: { "step-1": {...}, "step-2": {...} }
const combineStep = createStep({
  id: "combine",
  inputSchema: z.object({
    "step-1": z.object({ formatted: z.string() }),
    "step-2": z.object({ count: z.number() }),
  }),
  outputSchema: z.object({ result: z.string() }),
  execute: async ({ inputData }) => {
    return { result: `${inputData["step-1"].formatted} (${inputData["step-2"].count})` };
  },
});

export const workflow = createWorkflow({...})
  .parallel([step1, step2])
  .then(combineStep)
  .commit();

3. 条件分支 — .branch()

export const workflow = createWorkflow({...})
  .then(step1)
  .branch([
    [async ({ inputData: { value } }) => value > 10, highValueStep],
    [async ({ inputData: { value } }) => value <= 10, lowValueStep],
  ])
  .commit();

4. 数据映射 — .map()

export const workflow = createWorkflow({...})
  .then(step1)
  .map(async ({ inputData }) => {
    return { bar: `new ${inputData.foo}` };
  })
  .then(step2)
  .commit();

5. 嵌套工作流

// 工作流可作为步骤嵌入另一个工作流
export const parentWorkflow = createWorkflow({...})
  .then(childWorkflow)
  .commit();

控制流对比

方法 目的 输出 并发
.then(step) 顺序处理 步骤输出 一次一个
.parallel([a,b]) 对相同输入的不同操作 { a: U, b: V } 同时运行
.branch([...]) 条件路径选择 { selectedStep: U } 只执行一个分支
.map(fn) 数据转换 函数返回值 —

工作流状态

const step1 = createStep({
  id: "step-1",
  inputSchema: z.object({ message: z.string() }),
  outputSchema: z.object({ formatted: z.string() }),
  stateSchema: z.object({ counter: z.number() }),
  execute: async ({ inputData, state, setState }) => {
    console.log(state.counter);
    setState({ ...state, counter: state.counter + 1 });
    return { formatted: inputData.message.toUpperCase() };
  },
});

运行工作流

// 方式一：start — 等待完成
const run = await workflow.createRun();
const result = await run.start({ inputData: { message: "Hello" } });
if (result.status === "success") {
  console.log(result.result);
}

// 方式二：stream — 实时事件
const run = await workflow.createRun();
const stream = run.stream({ inputData: { message: "Hello" } });
for await (const chunk of stream.fullStream) {
  console.log(chunk);
}
const result = await stream.result;

结果类型

状态 唯一属性 描述
success result 工作流输出数据
failed error 导致失败的错误
suspended suspendPayload, suspended 暂停数据和已暂停步骤路径

暂停与恢复

工作流可在任何步骤暂停，用于收集额外数据、等待 API 回调、请求人工审批等。暂停时执行状态会保存为快照，可跨重启恢复。 

// 在步骤中暂停
const approvalStep = createStep({
  id: "approval-step",
  resumeSchema: z.object({ approved: z.boolean() }),
  execute: async ({ inputData, resumeData, suspend }) => {
    if (!resumeData?.approved) {
      return await suspend({ reason: "需要人工审批" });
    }
    return { result: `已审批通过` };
  },
});

// 恢复执行
const run = await workflow.createRun();
const result = await run.start({ inputData: {...} });
if (result.status === "suspended") {
  // 等待人工操作后恢复
  await run.resume({
    step: result.suspended[0],
    resumeData: { approved: true },
  });
}

Memory 记忆

LLM 是无状态的，Agent 需要通过 Memory 维护交互上下文。Mastra 提供三层记忆系统：

📜 消息历史
当前对话的消息列表，默认保留最近 10 条，提供短期上下文
🧠 工作记忆
跨交互持久化的"草稿本"，类似人类的活跃思维
🔍 语义召回
基于向量嵌入的相似性搜索，找回长对话中的相关消息

基本配置

import { Agent } from "@mastra/core/agent";
import { Memory } from "@mastra/memory";

export const myAgent = new Agent({
  id: "my-agent",
  name: "My Agent",
  instructions: "You are a helpful assistant.",
  model: "openai/gpt-5.1",
  memory: new Memory({
    options: {
      lastMessages: 20,              // 消息历史条数
      workingMemory: { enabled: true },  // 工作记忆
      semanticRecall: { topK: 3 },   // 语义召回
    },
  }),
});

消息历史

最简单的记忆形式，每个请求默认包含最后 10 条消息。可通过 lastMessages 调整。

查询消息

const storage = mastra.getStorage();
const memoryStore = await storage.getStore("memory");

// 分页查询单线程
const result = await memoryStore?.listMessages({
  threadId: "thread-123",
  page: 0,
  perPage: 50,
});

// 批量查询多线程
const multiResult = await memoryStore?.listMessages({
  threadId: ["thread-1", "thread-2"],
  page: 0,
  perPage: 100,
});

// 转换为 AI SDK 格式（用于 UI 渲染）
import { toAISdkV5Messages } from "@mastra/ai-sdk/ui";
const uiMessages = toAISdkV5Messages(result?.messages ?? []);

工作记忆

让 Agent 在交互之间保持关于用户的持久信息，类似于人类的"活跃思维"。

持久化范围

范围 说明 适用场景
resource（默认） 同一用户的所有线程共享 个人助理、客户服务
thread 每个线程独立隔离 不同主题的对话、临时会话

Markdown 模板模式

const memory = new Memory({
  options: {
    workingMemory: {
      enabled: true,
      scope: "resource",
      template: `# User Profile
- **Name**:
- **Location**:
- **Interests**:
- **Preferences**:
- **Long-term Goals**:`,
    },
  },
});

Zod Schema 模式（类型安全）

import { z } from "zod";

const userProfileSchema = z.object({
  name: z.string().optional(),
  location: z.string().optional(),
  preferences: z.object({
    communicationStyle: z.string().optional(),
    projectGoal: z.string().optional(),
  }).optional(),
});

const memory = new Memory({
  options: {
    workingMemory: {
      enabled: true,
      schema: userProfileSchema,  // 不要同时设置 template
    },
  },
});

模板 vs Schema

特性 模板（Markdown） Schema（Zod）
格式 自由文本块 结构化 JSON
更新方式 替换语义 — 每次提供完整内容 合并语义 — 仅提供更新字段
类型安全 无 有
适用场景 用户配置文件、草稿本 需要验证和编程访问的结构化数据

语义召回

基于 RAG 的搜索功能，帮助 Agent 在长对话中保持上下文。默认启用。

工作原理

新消息 → 查询向量数据库 → 相似性搜索 → 返回相关历史消息

配置参数

参数 默认值 说明
topK — 检索最相似的消息数量
messageRange — 每个匹配项前后包含的消息数
scope "resource" 搜索范围：resource 跨线程 / thread 当前线程

嵌入器配置

// 使用模型路由器（推荐）
import { ModelRouterEmbeddingModel } from "@mastra/core/llm";

const memory = new Memory({
  embedder: new ModelRouterEmbeddingModel("openai/text-embedding-3-small"),
});

// 或使用本地 FastEmbed（零外部依赖）
import { fastembed } from "@mastra/fastembed";
const memory = new Memory({ embedder: fastembed });

禁用语义召回

const memory = new Memory({
  options: { semanticRecall: false },  // 性能敏感场景可禁用
});

使用记忆的 Agent 调用

// 写入记忆
const response = await myAgent.generate("Remember my favorite color is blue", {
  memory: {
    resource: "user-123",      // 用户标识
    thread: "conversation-1",  // 会话标识
  },
});

// 读取记忆（相同的 resource + thread）
const response2 = await myAgent.generate("What's my favorite color?", {
  memory: {
    resource: "user-123",
    thread: "conversation-1",
  },
});
⚠️ 重要：每个线程在创建后其所有者 (resourceId) 无法更改。严禁为不同所有者重用相同的线程 ID，否则会导致查询错误。

支持的向量存储

RAG 检索增强生成

RAG 通过结合用户自己的数据源来增强 LLM 输出的准确性和可靠性。

流程

文档 → 分块 → 嵌入 → 存入向量库 → 相似性查询

代码示例

import { embedMany } from "ai";
import { PgVector } from "@mastra/pg";
import { MDocument } from "@mastra/rag";

// 1. 初始化文档
const doc = MDocument.fromText("Your document text here...");

// 2. 分块
const chunks = await doc.chunk({
  strategy: "recursive",
  size: 512,
  overlap: 50,
});

// 3. 生成嵌入
const { embeddings } = await embedMany({
  values: chunks.map((chunk) => chunk.text),
  model: new ModelRouterEmbeddingModel("openai/text-embedding-3-small"),
});

// 4. 存入向量数据库
const pgVector = new PgVector({
  connectionString: process.env.POSTGRES_CONNECTION_STRING,
});
await pgVector.upsert({ indexName: "embeddings", vectors: embeddings });

// 5. 查询
const results = await pgVector.query({
  indexName: "embeddings",
  queryVector: queryVector,
  topK: 3,
});

import { useChat } from "@ai-sdk/react";
import { DefaultChatTransport, type ToolUIPart } from "ai";

function ChatUI() {
  const { messages, sendMessage, status } = useChat({
    transport: new DefaultChatTransport({
      api: "http://localhost:4111/chat/weather-agent",
    }),
  });

  return (
    <div>
      {messages.map((msg) => (
        <div key={msg.id}>
          {msg.parts?.map((part, i) => {
            if (part.type === "text") {
              return <p>{part.text}</p>;
            }
            if (part.type?.startsWith("tool-")) {
              return (
                <div>
                  <pre>{JSON.stringify((part as ToolUIPart).input, null, 2)}</pre>
                  <pre>{JSON.stringify((part as ToolUIPart).output, null, 2)}</pre>
                </div>
              );
            }
            return null;
          })}
        </div>
      ))}
    </div>
  );
}

天气 Agent 完整实战

以下是在 skill-agent-react 项目中实现的天气 Agent 完整代码解析。

1. 天气工具 — weather-tool.ts

import { createTool } from "@mastra/core/tools";
import { z } from "zod";

export const weatherTool = createTool({
  id: "get-weather",
  description: "Get current weather for a location",
  inputSchema: z.object({
    location: z.string().describe("City name"),
  }),
  outputSchema: z.object({
    temperature: z.number(),
    feelsLike: z.number(),
    humidity: z.number(),
    windSpeed: z.number(),
    windGust: z.number(),
    conditions: z.string(),
    location: z.string(),
  }),
  execute: async ({ context }) => {
    const { location } = context;

    // Step 1: Geocoding — 城市名 → 经纬度
    const geoUrl = `https://geocoding-api.open-meteo.com/v1/search?name=${encodeURIComponent(location)}&count=1`;
    const geoRes = await fetch(geoUrl);
    const geoData = await geoRes.json();
    const { latitude, longitude, name } = geoData.results[0];

    // Step 2: Forecast — 获取实时天气
    const weatherUrl = `https://api.open-meteo.com/v1/forecast?latitude=${latitude}&longitude=${longitude}¤t=temperature_2m,relative_humidity_2m,apparent_temperature,wind_speed_10m,wind_gusts_10m,weather_code`;
    const weatherRes = await fetch(weatherUrl);
    const weatherData = await weatherRes.json();
    const current = weatherData.current;

    // Step 3: WMO 天气码 → 可读字符串
    const conditions = getWeatherCondition(current.weather_code);

    return {
      temperature: current.temperature_2m,
      feelsLike: current.apparent_temperature,
      humidity: current.relative_humidity_2m,
      windSpeed: current.wind_speed_10m,
      windGust: current.wind_gusts_10m,
      conditions,
      location: name,
    };
  },
});

3. 天气 Workflow — weather-workflow.ts

import { createWorkflow, createStep } from "@mastra/core/workflows";
import { z } from "zod";
import { weatherAgent } from "../agents/weather-agent";

// Step 1: 获取天气预报
const fetchWeather = createStep({
  id: "fetch-weather",
  inputSchema: z.object({ city: z.string() }),
  outputSchema: z.object({
    maxTemp: z.number(),
    minTemp: z.number(),
    precipitationChance: z.number(),
    condition: z.string(),
  }),
  execute: async ({ inputData }) => {
    // 调用 Open-Meteo API 获取当日天气...
    return { maxTemp: 28, minTemp: 18, precipitationChance: 30, condition: "Partly Cloudy" };
  },
});

// Step 2: 规划活动
const planActivities = createStep({
  id: "plan-activities",
  inputSchema: z.object({
    maxTemp: z.number(),
    minTemp: z.number(),
    precipitationChance: z.number(),
    condition: z.string(),
  }),
  outputSchema: z.object({ activities: z.string() }),
  execute: async ({ inputData }) => {
    const prompt = `Based on: max ${inputData.maxTemp}°C, min ${inputData.minTemp}°C, ${inputData.precipitationChance}% rain, ${inputData.condition}. Plan morning/afternoon/indoor activities.`;
    const stream = await weatherAgent.stream(prompt);
    let activities = "";
    for await (const chunk of stream.textStream) {
      activities += chunk;
    }
    return { activities };
  },
});

export const weatherWorkflow = createWorkflow({
  id: "weather-workflow",
  inputSchema: z.object({ city: z.string() }),
  outputSchema: z.object({ activities: z.string() }),
})
  .then(fetchWeather)
  .then(planActivities)
  .commit();
