## 2. 入职第一天

### 2.1 必办事项

1. **领取办公设备**：到行政部领取笔记本电脑（MacBook Pro M3）、显示器、机械键盘
2. **开通系统权限**：IT 部门会为你开通以下账号：
   - 企业邮箱（@company.com）
   - GitLab 代码仓库（gitlab.company.com）
   - 飞书（即时通讯和 OA）
   - 内部 Wiki（wiki.company.com）
3. **完成入职培训**：在线学习平台完成以下必修课程（入职首周内完成）
   - 公司文化与制度（约 30 分钟）
   - 信息安全规范（约 45 分钟）
   - 开发环境搭建指南（约 1 小时）

### 2.2 认识你的导师

每位新员工都会分配一位入职导师（Buddy），通常是你所在团队的高级工程师。导师的职责：

- 帮助你熟悉代码仓库和开发流程
- 解答技术问题，带你熟悉技术栈
- 每周进行一次 1v1 沟通，了解你的适应情况
- 入职满月时协助完成试用期第一次评估

### 2.3 开发环境搭建

推荐的开发环境配置：

- **操作系统**：macOS Sonoma 14.0+ 或 Ubuntu 22.04+
- **IDE**：VS Code（推荐安装团队共享的 settings.json）
- **Node.js**：v20 LTS（使用 nvm 管理）
- **Python**：3.11+（AI 平台组需要）
- **数据库**：Docker 运行 PostgreSQL 15 + Redis 7
- **包管理器**：pnpm（前端项目）、npm（后端项目）

```bash
# 安装 nvm
curl -o- https://raw.githubusercontent.com/nvm-sh/nvm/v0.39.0/install.sh | bash
nvm install 20
nvm use 20

# 安装 pnpm
npm install -g pnpm

# 克隆主要代码仓库
git clone git@gitlab.company.com:platform/ai-agent-framework.git
git clone git@gitlab.company.com:frontend/web-app.git
```

## 4. 薪酬与福利

### 4.1 薪酬结构

- **月薪**：每月 15 日发放上月工资（遇到节假日顺延到最近工作日）
- **年终奖**：通常为 2-4 个月薪资，根据个人绩效和公司业绩确定
- **期权**：入职满 1 年且绩效达标者可参与期权授予计划
- **薪资保密**：公司实行薪资保密制度，请勿互相打探薪资信息

### 4.2 五险一金

- 按深圳市最高基数缴纳
- 养老保险、医疗保险、失业保险、工伤保险、生育保险
- 住房公积金：个人 5% + 公司 12%
- 补充商业保险：入职即享，覆盖意外和重疾

### 4.3 其他福利

- **餐饮**：公司食堂午餐补贴 20 元/天，园区周边有多个商圈
- **交通**：每月交通补贴 500 元（地铁/公交/打车均可）
- **体检**：每年一次全面体检（三甲医院）
- **团建**：每季度一次团队活动（预算人均 500 元）
- **节日礼品**：春节、端午、中秋发放节日礼品
- **生日福利**：生日当月享受半天生日假 + 生日礼物
- **学习基金**：每年 5000 元学习基金（可用于购买书籍、课程、技术大会门票）

### 4.4 面试报销

入职后可报销面试产生的交通费用，需提供发票或电子票据，在飞书 OA 中提交报销申请，审批通过后随下月工资发放。

## 5. 技术规范

### 5.1 代码规范

- **TypeScript 项目**：遵循 ESLint + Prettier 配置，提交前必须通过 lint 检查
- **Git 提交规范**：使用 Conventional Commits 格式

```
feat: 新增用户注册功能
fix: 修复登录页面样式错位问题
docs: 更新 API 文档
refactor: 重构数据库连接池模块
test: 补充用户模块单元测试
chore: 升级依赖版本
```

- **Code Review**：每个 MR 至少需要 1 位同事 Review 通过后才能合并
- **分支策略**：
  - `main`：生产分支，受保护
  - `develop`：开发分支
  - `feature/*`：功能分支，从 develop 创建
  - `hotfix/*`：紧急修复分支，从 main 创建

### 5.2 部署流程

```
feature 分支 → 提交 MR → Code Review → 合并到 develop
→ 自动部署到 staging 环境 → QA 测试 → 发布审批
→ 合并到 main → 自动部署到生产环境
```

- staging 环境：staging.company.com
- 生产环境：www.company.com
- 部署工具：GitLab CI/CD + Kubernetes

### 5.3 日常站会

- **时间**：每天上午 10:15，15 分钟以内
- **形式**：围绕白板或飞书文档
- **内容**：每人回答三个问题
  1. 昨天做了什么？
  2. 今天计划做什么？
  3. 有没有遇到阻碍？

### 5.4 需求评审

- 每两周一次 Sprint 规划会（周三下午）
- 产品经理提前 2 天在飞书文档中整理需求
- 技术评审在需求评审后 1 天内完成
- 评审内容包括：技术可行性、工期评估、风险点

## 6. IT 与安全

### 6.1 VPN 使用

- 访问公司内网需要连接 VPN（OpenVPN）
- VPN 配置文件在 IT Wiki 中下载
- VPN 账号与飞书账号一致
- 如遇连接问题，联系 IT 支持（@it-support）

### 6.2 信息安全红线

以下行为严格禁止，违反可能导致解除劳动合同：

1. 将生产环境数据库密码、API 密钥等敏感信息提交到公开代码仓库
2. 将公司源代码上传到个人 GitHub 账号或任何外部平台
3. 未经授权访问或导出用户数据
4. 在公共场合讨论公司未公开的技术方案或业务数据
5. 使用未经审批的第三方 SaaS 服务处理公司数据

### 6.3 密码管理

- 所有系统密码使用公司统一密码管理器（1Password Teams）
- 数据库、API 密钥等敏感信息通过环境变量或密钥管理服务（Vault）管理
- 代码中禁止硬编码任何密码或密钥
- 建议每 90 天更换一次个人密码

### 6.4 数据备份

- 代码仓库由 GitLab 统一管理，每日自动备份
- 数据库每日全量备份 + 实时增量备份
- 备份保留策略：最近 30 天每日备份 + 最近 12 个月月度备份

## 8. 试用期说明

### 8.1 试用期时长

- 初级工程师：3 个月
- 中高级工程师：2 个月
- 架构师及以上：1 个月

### 8.2 试用期目标

入职导师会和你一起制定试用期目标，通常包括：

1. **第一个月**：熟悉代码仓库和开发流程，独立完成 2-3 个小需求
2. **第二个月**：独立负责一个完整功能模块，参与 Code Review
3. **第三个月（如适用）**：主导一个小型项目，展示技术能力和协作能力

### 8.3 试用期评估

- 评估时间：试用期结束前 1 周
- 评估人：直属主管 + 入职导师
- 评估维度：技术能力、协作能力、学习态度、文化契合度
- 结果：通过 / 延长试用期（最多 1 个月） / 不通过

## 9. 常见问题

### Q：公司用什么技术栈？

A：后端以 Node.js/TypeScript 为主，前端 React + TypeScript，AI 平台使用 Python。数据库主要用 PostgreSQL 和 Redis。基础设施基于 Kubernetes。

### Q：加班多吗？

A：我们推崇高效工作，不鼓励无效加班。项目冲刺期可能会有适度加班，加班可申请调休或按加班费标准发放。

### Q：有技术培训机会吗？

A：有的。每年 5000 元学习基金，公司定期组织技术分享和读书会。表现优秀者可申请参加外部技术大会（公司报销）。

### Q：电脑坏了怎么办？

A：飞书联系 @it-support，办公时间内通常 30 分钟内响应。

### Q：如何申请调薪？

A：每年有两次调薪窗口（3 月和 9 月），主管会根据绩效评估结果提出调薪建议。特殊情况下（如晋升、市场调整）也可申请额外调薪。

### Q：公司有没有零食？

A：有的！茶水间有免费咖啡、茶、零食，每周三下午还有下午茶。冰箱可以存放个人食物。
